1.环境准备

阿里云主机一台（1 vCPU，2 GiB，有公网IP）

1.1 开放MySQL 3306 端口：

更多 -> 网络和安全组 -> 安全组配置 -> 配置规则 -> 添加安全组规则

端口范围选择 3306/3306

授权对象选择 0.0.0.0/0

2.下载安装

登录官网https://dev.mysql.com/downloads/repo/yum/

2.1 获取 yum 源

2.2 添加 yum 源

2.3 选择安装版本 [可选]

默认安装最新版 MySQL ，如果希望下载旧版，如5.7，则需要修改源设置

找到

[mysql57-community]
name=MySQL 5.7 Community Server
baseurl=http://repo.mysql.com/yum/mysql-5.7-community/el/7/$basearch/
enabled=0
gpgcheck=1
gpgkey=file:///etc/pki/rpm-gpg/RPM-GPG-KEY-mysql

将 enabled = 0 改成 enabled = 1

再找到

[mysql80-community]
name=MySQL 8.0 Community Server
baseurl=http://repo.mysql.com/yum/mysql-8.0-community/el/7/$basearch/
enabled=1
gpgcheck=1

将 enabled = 1 改成 enabled = 0

检查是否安装成功

若出现

mysql-connectors-community/x86_64 MySQL Connectors Community 51
mysql-tools-community/x86_64 MySQL Tools Community 63
mysql57-community/x86_64 MySQL 5.7 Community Server 267

则代表已经切换成 5.7 版本了

4.开放外界访问

4.1 修改用户

4.1.1 修改用户密码

第一次启动MySQL后，日志文件中会给root用户随机生成一段密码，我们需要获得这个密码，接下来需要使用

运行结果
2019-06-20T04:40:45.020689Z 5 [Note] [MY-010454] [Server] A temporary password is generated for root@localhost: i?g3?toJu3hk

最后面的 i?g3?toJu3hk 就是随机密码

4.1.2 登录root用户

提示需要输入密码，此时输入刚才获得的密码

4.1.3 设置新密码

新密码 设置要求：大写字母 + 小写字母 + 数字 + 特殊符号

4.1.4 配置生效

4.2 创建用户

4.2.1 创建可在任何主机登录的用户

参数 三种选择：

  • Localhost：用户只能在本机登录
  • %：通配符，用户可在任何 IP 主机登录
  • IP地址：用户只能从指定 IP 的主机登录，IP 可使用通配符 % ，如 192.168.100.%

TIP：当需要把一个用的 <host> 设置为 % 时，必需同时创建一个和该用户拥有相同用户名和密码，相同数据库权限的 localhost 的用户，否则该用户可能无法在 localhost 登陆 MySQL

4.2.2 赋予用户访问权限

针对指定用户，授予访问指定数据库和指定表

参数 可设置为：

  • ALL：表示所有权限
  • SELECT / INSERT / UPDATE 等指定权限，多权限用，隔开
  • database 和 table 可以设置为 *，表示所有数据库、所有表

4.2.3 撤销用户访问权限

5.可能出现的问题

5.1 Navicat连接报错

报错描述：

使用 Navicat 连接数据库，报错

错误原因：

mysql8.0 引入了新特性 caching_sha2_password；这种密码加密方式Navicat 12以下客户端不支持

Navicat 12以下客户端支持的是 mysql_native_password 这种加密方式

解决方案：

查询结果

可以看到加密方式都为 caching_sha2_password

可以使用命令修改为 mysql_native_password 加密模式
