项目启动思路

考虑到前端项目和后端项目需要在同一个环境下，避免跨域问题，以及都要放置在docker的容器上，因此考虑采用 docker-compose 统一配置，这样前端的请求就可以能成功到达后端

安装Docker-Compose

环境需要安装 py-pip , python-dev (Centos中为python-devel), libffi-dev , openssl-dev , gcc , libc-dev

编辑nginx.conf

这里因为主域名 www.confusedbear.cn 和子域名 tl.confusedbear.cn 对应两个不同的项目，因此采用了两个 server 分别处理

其中值得注意的是：

如果缺少 include，会导致 antd 的样式全部丢失

如果缺少 try_files，会导致路由设置为 browser history 的 React 项目刷新或跳转就会丢失页面

详情可以参考： https://www.jianshu.com/p/51ba2bec00c7

这里的作用就是将对 [主域名]/task 开始的所有请求，转移到后端项目的 /user 路径上，只有这样设置，前端的请求才可以和后端连接上
